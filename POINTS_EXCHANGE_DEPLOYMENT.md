## ⚠️ 重要注意事项

### 1. 必须先执行SQL脚本
如果不执行SQL脚本，会出现以下错误：
- `Table 'points_gift' doesn't exist`
- `Table 'points_exchange' doesn't exist`

### 2. 积分不足时的提示
当学生积分不足时：
- 系统会弹出提示框
- 显示当前积分和所需积分
- 不会扣除积分
- 不会创建兑换记录

### 3. 兑换后积分立即扣除
- 兑换成功后积分立即减少
- 请在兑换前确认积分充足
- 可以在"我的积分"页面查看积分明细
- **谨慎操作，兑换后无法撤销**

### 4. 库存为0时无法兑换
- 当奖品库存为0时，"立即兑换"按钮仍然显示
- 但点击后会提示"奖品库存不足"
- 需要管理员补充库存或下架该奖品

### 5. 兑换记录状态说明
- **待领取（0）**：刚兑换，还未领取奖品
- **已领取（1）**：学生已线下领取奖品
- **已取消（2）**：兑换被取消（暂未实现取消功能）

## 🐛 常见问题排查

### 问题1：访问积分商城403错误
**原因**：Security权限未配置或未重启应用  
**解决**：
1. 检查 `SecurityConfig.java` 是否包含积分兑换权限配置
2. 重新编译并重启应用

### 问题2：兑换时提示"学生信息不存在"
**原因**：学生表中没有对应用户ID的记录  
**解决**：
```sql
-- 检查学生信息
SELECT * FROM sys_student WHERE user_id = [用户ID];
```

### 问题3：奖品列表为空
**原因**：数据库中奖品表为空或所有奖品都下架了  
**解决**：
```sql
-- 检查奖品数据
SELECT * FROM points_gift WHERE status = 1 AND stock > 0;

-- 如果没有数据，重新执行SQL脚本
```

### 问题4：评价后没有自动加分
**原因**：积分规则未配置或PointsService有问题  
**解决**：
```sql
-- 检查积分规则是否存在
SELECT * FROM points_rule WHERE rule_code LIKE 'EVALUATION_%';

-- 检查积分明细
SELECT * FROM student_points WHERE student_id = [学生ID] 
ORDER BY create_time DESC LIMIT 10;
```

### 问题5：Thymeleaf模板解析错误
**原因**：模板语法错误或变量未传递  
**解决**：
1. 检查控制台错误日志
2. 确认Controller中添加了所有必需的model属性
3. 检查模板中的Thymeleaf表达式语法
